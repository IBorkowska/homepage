Zasady do zapamiętania

1. Nie wszystkie zasady w CSS są dziedziczone.


Składnia CSS

selektor {właściwość: wartość;}


Gramatyka CSS

  • piszemy małymi literami
  • średnik na końcu instrukcji
  • dwukropek rozdziela własciwość od wartości
  • komentarz w CSS /* */ - skrót CTRL + slash
  • nazwy właściwości piszemy z myślnikiem
  • atrybut class może mieć przypisaną więcej niż jedną klasę
  • p.red.big - czytamy weź element p, który ma jednocześnie klasę red i big
  • RESET: * {margin:0; padding: 0;}

3 sposoby dodawania stylów

  • liniowo - bezpośrednio w elemencie - <p style="font-size: px; color:red;"> UWAGA! Style liniowe są najważniejsze, mają najwyższy priorytet. Z tego sposobu korzystamy w JS i wtedy ten rodzaj dodawania stylów jest najbardziej przydatny
  • sekcji <head> w znacznikach <style> </style> Sposób ten warto stosować w trakcie nauki CSS. UWAGA! Może być więcej niż jeden znacznik style w head. Może też pojawić się w sekcji body.
  • w zewnętrznym pliku - w znaczniku <link> I to jest najlepszy sposób.

Nazewnictwo klas

  • używaj pojedynczych słów z j. angielskiego
  • nazywaj rzeczy zgodnie z ich funkcją na stronie
  • startuj z małej litery np. header nie Header
  • nie zaczynaj od cyfry
  • jeśli musisz użyć dwóch słów użyj tzw. kebab case np.: news-listing
  • staraj się, żeby element miał jedną klasę (nie zwiększaj specyficzności bez potrzeby
  • specyficzność - klasyfikacja medalowa
  • html class="name" = css .name
  • html id="name" = css #name
  • stylujemy po klasach (bez zagnieżdżeń!)
  • nazywamy klasy po angielsku zgodnie z przeznaczeniem
  • tworzymy klasy nawet mamy jeden element danego typu
  • specyficzność może dać popalić ;)
  • nie używamy !important i stylów inline'owych
